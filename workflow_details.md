# Workflow Details

SciFigure uses a controller-centered workflow.

The controller first writes a task file. It then writes a style summary and an asset decision table. Image assets are created only after this decision table exists. The controller selects the backend according to the current operating system and writes the slide generation code. A preview image is exported and checked before final delivery.

Default review points:

1. task understanding
2. asset decision
3. first preview
4. final delivery

The final slide should remain editable. Titles, labels, text, panels, arrows, axes, and formulas are not treated as decorative images.
